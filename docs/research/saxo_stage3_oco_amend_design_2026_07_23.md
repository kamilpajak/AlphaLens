## Chosen approach (HYBRID)
HYBRID = Approach 1's minimal surface (reuse the existing `UpgradeToOco` action for B0; ONE new `AmendStop` action; delete only the dead arm-C emission) + Approach 3's absolute-target-to-live-owned PATCH semantics and explicit rung-1 REFUSE, hardened by EVERY FIXABLE adversary finding. Three write paths: (B0) OCO-direct on a truly-naked fresh fill via `UpgradeToOco(supersede_ids=())`; (AmendStop) an in-place PATCH resize of a SINGLE clean standalone stop, used to grow-up (composes with B1) and to converge arm-A over-hedge DOWN (the #884 gap); (rung-1) a resting standalone stop when OCO is enabled stays stop-only forever (arm-C upgrade deleted). Two new dark env flags (`ALPHALENS_BROKER_OCO_ENABLED` repurposed to gate B0; new `ALPHALENS_BROKER_AMEND_ENABLED`), PATCH wired into the POST-safe never-blind-retry lane with idempotent=False. No BROKEN path is adopted: the rung1->2 upgrade (2x-owned, proven-live-rejected) is REJECTED entirely; PositionId reduce-only (HTTP-400-refuted) is not attempted; siblings-array OCO body is not used.

## Rationale
Approach 1 is the smallest diff over proven Stage-1.5 machinery, but its two seams were broken by adversaries: (H1/A1/A2) B0's fallback-on-ANY-BrokerError double-commits when the OCO write is ambiguous-but-landed or when B0 re-fires on a list-orders-lagged empty-legs view; (H2) the grow arm dropped the partial-fill guard; (H3/H5/A3) the amend ref gen-keyed-on-qty collides with Saxo's 15s dedup and the `-stop-` POST namespace; (verdict-2) grow-reject leaves a naked delta with only a soft alert, the down-clamp is one-directional (naked when owned grows mid-tick), and a benign fill-race 400 falsely latches capability off. Approach 3's absolute-target insight is the key unlock: re-applying "set Amount = live owned" is idempotent-in-effect (two sets = owned, never 2x), so a cross-tick amend re-emit is safe and the ref can be MONOTONIC (never dedup-swallow a needed resize) while a single write stays idempotent=False (never blind-retry ambiguous 5xx). Every adversary finding is folded in as an explicit mitigation (below), so the residual windows are all bounded to <=1 poll interval (or an over-COVERED, safe-direction state) and self-heal by reconciling against live broker state next tick. The rung-1 REFUSE is chosen because no mechanical transition of a resting lone stop into an OCO pair is both never-naked AND never-double-commit (PATCH cannot add a TP leg; cancel-then-OCO is naked; OCO-then-cancel is 2x-owned rejected live) — the only cost is TP capture on the pre-OCO cohort, an upside/lens concern, not a downside-safety one; option X reaches OCO only at the fresh-fill moment and the system converges by turnover.

## rung-1 transition = REFUSE
CHOSEN BEHAVIOUR: a position that ALREADY has a resting rung-1 standalone stop when OCO is enabled STAYS STOP-ONLY for its entire life — it is never upgraded to OCO. Implementation = DELETE the arm-C rung1->2 emission at position_manager.py:496-507 (the `UpgradeToOco(..., supersede_ids=_stop_leg_ids(legs))` block AND the grow-after-OCO guard at :494 collapse into it). Arm C's tail becomes: after the healthy `tp_qty + _QTY_EPS >= owned -> return [NoOp()]` branch (:479), replace the remaining `if plan.tp_price is None or ... -> NoOp` + guard + UpgradeToOco with a single `return [NoOp()]` carrying the comment "rung1->2 conversion of a resting standalone stop is unsafe by construction (Stage 3): PATCH cannot add a TP leg, cancel-then-OCO is naked, OCO-then-cancel is 2x-owned rejected live — refuse, stay stop-only; OCO is reached only via B0 on a fresh naked fill." The `UpgradeToOco` action and `_execute_upgrade_to_oco` executor are KEPT (B0 is now their sole emitter, always supersede_ids=()). This choice is the only one preserving all three invariants: the resting stop is never touched, no OCO is placed atop it (so the 2x-owned SellOrdersAlreadyExist state can never arise), no naked window opens. It also removes a self-inflicted permanent-degrade path — the old arm-C emission, with a resting rung-1 stop, would place-OCO-first -> 2x owned -> BrokerError -> permanent `oco_unsupported`. Invariant cost is a CAPABILITY limitation, not a safety hole: positions that filled before OCO was enabled (or during any OCO outage that placed a fallback stop, or on a uic latched oco_unsupported) get no automated take-profit and exit via the deep-OTM disaster stop or a manual/next-session close. The system converges to full OCO coverage purely by turnover — every NEW fill under OCO-enabled takes B0, and the stop-only residue drains as pre-OCO positions close.

## Bounded windows (residual, self-healing)
- FRESH-FILL naked window on an AMBIGUOUS B0 OCO write: <=1 poll interval (~45s). Self-heal: no inline fallback is placed (would double-commit if the OCO landed); the next tick reconciles against live broker state — OCO visible -> arm C covered; OCO landed-but-list-lagged -> B0 re-POST -> SellOrdersAlreadyExist -> benign 'already covered'; OCO not landed -> B0 re-POST -> 2xx covers. This is the pre-existing fresh-fill first-protect window; option X does not widen it.
- GROW delta under-cover while a PATCH amend is in flight or after an amend reject: <=1 poll interval. Self-heal: absolute-target amend to LIVE owned; on amend reject, amend_failed is journaled -> amend_recently_failed suppresses amend next tick -> B1 additive-delta places a second stop covering the delta. Never permanently naked; covered by a proven primitive within one tick.
- oco_recently_placed SUPPRESSION window if a just-placed OCO is EXTERNALLY cancelled before list-orders ever shows its legs: <= _OCO_PLACED_TTL_S (120s). Self-heal: after the TTL the marker expires and B0 re-fires to re-cover. Residual is documented and SIM-only (the daemon is the sole manager; no external cancellation path in normal operation); the moment any leg appears, B0's `not legs` gate is false regardless of the marker.
- MULTI-TIER over-hedge downsize (2+ stops from B1, or an OCO leg present): NOT converged by AmendStop (scoped to a single clean standalone stop) — falls to the unchanged #884 place-residual-first arm A, which is OVER-COVERED, never naked (safe direction). No naked window; the over-sized stop persists until Q9 (OCO/multi-order in-place amend) lands. Saxo NotOwned is the fire-time backstop.
- amend_recently_failed retry window: amend is disabled for a uic for <= _AMEND_FAILED_TTL_S (120s) after a failure; during it B1 additive (grow) / place-first (downsize) fully cover the position, so there is NO naked window — only a temporary loss of the in-place-amend optimization, which self-clears and retries after the TTL (so a benign fill-race 400 is not a permanent latch).
