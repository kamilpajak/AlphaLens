## Chosen approach
Proposal 1 — EXTEND the existing Stage-3 AmendStop rail with a second selector (`_oco_stop_leg`), NOT a new `AmendOcoLeg` action. Add one pure selector plus two guarded amend branches (grow in arm B, downsize in arm A) plus one propagation-lag NoOp guard, all in position_manager.py. The `AmendStop` dataclass, the `_exit_amend_ref`/`next_amend_seq` monotonic ref lane, the `amend_recently_failed` TTL fold, the `_amend_enabled()` env gate, the control_loop dispatch (control_loop.py:1312), and the executor `_execute_amend_stop` (control_loop.py:1459-1527) are reused BYTE-FOR-BYTE. Ships DARK behind the existing `ALPHALENS_BROKER_AMEND_ENABLED` (no new flag). Builds ON TOP of PR #894 (Stage 3, currently dark on the stage35-read worktree, 778d1023) — must land after #894 merges to main or be stacked on it.

## Rationale
Proposal 1 is strictly less surface than Proposal 2 for identical behavior. VERIFIED against the actual Stage-3 code in the worktree: `_execute_amend_stop` (control_loop.py:1459-1527) reads ONLY the AmendStop dataclass fields, re-reads live owned via `get_positions_by_uic` and clamps target absolute (:1484-1485), then calls `amend_stop(uic, order_id, side, order_type, target, stop_price, request_id)` (:1496-1504) — nothing branches on leg shape, so an OCO child stop's order_id PATCHes through unchanged and Q9 (established ground truth: symmetric propagation to the Limit sibling) resizes the whole pair Saxo-side. The AmendStop action already carries every field an OCO-leg amend needs (uic, side, order_id, order_type, absolute target_qty, stop_price, request_id, reason). Proposal 2's new `AmendOcoLeg` type would force edits to the Action union (:244), `__all__` (:734), a control_loop import, a dispatch isinstance-tuple, and an executor type-hint widen — for a marginal telemetry gain that Proposal 1 already delivers via a distinct `reason=` string ("grow-after-OCO ..." / "OCO downsize ..."). Both adversaries rated both paths FIXABLE/medium with the SAME two real findings (downsize propagation-lag teardown; mid-fill TOCTOU), so surface minimality decides. Every FIXABLE finding is folded in as an explicit mitigation below; no path was rejected as BROKEN. Gate on `_amend_enabled()` ONLY (not also `_oco_enabled()`): a resting OCO pair exists only because OCO was enabled at B0-placement time, and resizing it in place is strictly safer (fewer resting orders, TP-capture preserved) than the B1-additive/place-residual fallback even if OCO was since disabled.

## Selector change (_oco_stop_leg)
NEW pure fn `_oco_stop_leg(legs: tuple[OrderState, ...]) -> OrderState | None`, added adjacent to `_sole_standalone_stop` (right after position_manager.py:423). It is the OCO-REQUIRING inverse of `_sole_standalone_stop` (whose guards 2 `if _is_oco_leg(stop): return None` @415 and 3 `if any(leg.order_type in TP_TYPES ...): return None` @417 are load-bearing for the standalone rail and MUST NOT be relaxed in place). Body (cleaner len==2 form — a single OCO pair is the sole coverage because SellOrdersAlreadyExist guarantees at most one pair per uic, and len==2 auto-rejects any stray leg alongside): `if len(legs) != 2 or not all(_is_oco_leg(l) for l in legs): return None` — guard 1 (exactly two legs, both OCO; recognises a Q7-asymmetric pair because `_is_oco_leg` is OR of OrderRelation=='Oco' and the -oco- ref infix); `stops=[l for l in legs if l.order_type in STOP_TYPES]; tps=[l for l in legs if l.order_type in TP_TYPES]; if len(stops)!=1 or len(tps)!=1: return None` — guard 2 (well-formed {one StopIfTraded, one Limit} pair); `if (stops[0].filled_quantity or 0.0) > _QTY_EPS or (tps[0].filled_quantity or 0.0) > _QTY_EPS: return None` — guard 3 (NEITHER leg partially triggered — mitigation for the mid-fill TOCTOU: a partially-filled pair defers to the partial-fill-aware place-residual-first arm because Q9 propagation onto a partially-filled leg is UNPROVEN); `return stops[0]` (the OCO child stop whose Amount the raw PATCH resizes; Q9 propagates to the Limit sibling). Uses only existing primitives (STOP_TYPES, TP_TYPES, _is_oco_leg, _leg_amount, _QTY_EPS).

## Reconcile arm changes
THREE insertions in `_reconcile_long`, all gated on `_amend_enabled()` so the feature is fully DARK (byte-identical behavior when the flag is off). (1) GROW — arm B DOWNSIDE DEFICIT (`if stop_qty + _QTY_EPS < owned:` @576), insert AFTER the standalone grow-amend if-block (ends :638) and BEFORE the (B1) additive block (:650): `oco_stop = _oco_stop_leg(legs); if (_amend_enabled() and oco_stop is not None and uic not in view.oco_unsupported and uic not in view.amend_recently_failed and _leg_amount(oco_stop) + _QTY_EPS < owned): return [AmendStop(uic, _SIDE, oco_stop.order_id, oco_stop.order_type or "StopIfTraded", owned, plan.stop_price, _exit_amend_ref(plan.entry_crid, plan.next_amend_seq()), reason="grow-after-OCO — PATCH OCO stop leg up in place")]`. B0 (:585-611) is skipped for a resting OCO because `not legs` is False; the standalone grow-amend (`_sole_standalone_stop`) returns None because a TP leg is present — so this OCO-grow amend is the first arm that can fire. (2) DOWNSIZE — arm A OVER-HEDGE (`if total > owned + _QTY_EPS:` @516), insert AFTER the standalone downsize-amend if-block (ends :543) and BEFORE `bad = _group_with_partial_fill(legs) or _newest_group(legs)` (:544): `oco_stop = _oco_stop_leg(legs); if (_amend_enabled() and oco_stop is not None and uic not in view.oco_unsupported and uic not in view.amend_recently_failed and _leg_amount(oco_stop) > owned + _QTY_EPS): return [AmendStop(uic, _SIDE, oco_stop.order_id, oco_stop.order_type or "StopIfTraded", owned, plan.stop_price, _exit_amend_ref(plan.entry_crid, plan.next_amend_seq()), reason="OCO downsize — PATCH OCO stop leg down in place")]`. (3) MITIGATION M1 — OCO propagation-lag NoOp guard, inserted IMMEDIATELY AFTER insertion (2) and BEFORE `bad = ...` (:544): `if _amend_enabled() and oco_stop is not None: return [NoOp()]`. Rationale: reaching here with `oco_stop is not None` means the pair is a clean unfilled OCO whose stop leg is already <= owned (else the downsize amend fired) yet total>owned — i.e. `_sell_commitment`'s max-over-group is the TP leg still showing the OLD larger Amount because list-orders lags Q9's symmetric propagation. Tearing it down via place-residual-first would cascade-cancel the pair the previous tick just resized, on a transient read lag. The downside is fully covered (stop==owned); NoOp one tick until the TP read catches up (then total==owned -> arm C NoOp @678-682). Gating the guard on `_amend_enabled()` keeps arm A byte-identical when dark. AmendStop docstring (:224-232) widened from "SINGLE clean resting standalone stop" to also name the resting-OCO-pair child stop; NO new field. `order_type or "StopIfTraded"` (NOT the standalone sites' `or ""` @537/632) because a naive /orders/me read can echo OrderType:None (Stage-3.5 write-side gotcha) — the mapped OrderState reads OpenOrderType so it is usually populated, defaulting is belt-and-suspenders. Steady-state healthy OCO {stop=owned,tp=owned} trips NEITHER amend arm nor the lag guard (arm A no: commitment==owned; arm B no: stop_qty==owned; arm C: tp_qty>=owned -> NoOp).

## Bounded windows
- Decision->execute TOCTOU for a TP-sibling partial fill: bounded to WITHIN ONE ~45s poll tick (the gap between build_protection_view's list_working_sell_orders snapshot and the PATCH landing, sub-second-to-seconds). Selector guard 3 requires BOTH OCO legs unfilled at decision; the executor re-reads and clamps owned; Saxo NotOwned (Q3a, cash account cannot short) bounds the worst case to a REJECT, never a completed short. Enablement additionally gated on the Q10 probe.
- Downsize propagation-lag NoOp (M1): bounded to 1-2 ticks — Q9 propagation is immediate + symmetric Saxo-side (established ground truth); our list-orders cache refreshes each tick, so the lagging TP read converges within one refresh. Downside is FULLY covered (stop==owned) for the entire lag; the only residual is a latent TP-oversell risk, NotOwned-capped. A persistent lag NoOp (>N ticks) is a monitoring concern (see open questions), not an invariant break.
- Amend-fail TTL fold: one TTL window after a PATCH reject; the next tick's OCO-amend guard (`uic not in amend_recently_failed`) fails, dropping grow to B1 additive (2nd standalone stop for the delta -> over-covered) and downsize to place-residual-first (residual stop placed BEFORE the OCO stop is superseded -> over-covered). No permanent latch (TTL self-clears; a benign fill-race 400 retries).
- Additive-ref 15s dedup: NOT applicable to the OCO amend — it uses the monotonic `_exit_amend_ref(plan.entry_crid, plan.next_amend_seq())` lane (always max+1), NOT the qty-keyed `next_gen` lane that B1 additive uses, so opposite grow/downsize resizes to the same qty across ticks never collide in Saxo's 15s request-id window.

- order_type default back-port scope: the two existing standalone AmendStop sites pass `sole.order_type or ""` (:537/632). - amend-fail non-convergence clutter is accepted, not fixed: after a downsize amend-fail, a SellOrdersAlreadyExist rejection on the residual PlaceStop (Saxo refusing a 3rd sell order while the OCO rests) leaves the pair over-covered permanently until turnover (never naked, NotOwned-capped). Confirm this is acceptable rather than adding a dedicated OCO-teardown-then-replace path.
