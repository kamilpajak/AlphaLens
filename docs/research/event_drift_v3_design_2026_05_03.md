## Thesis

Foster (1977) standardised unexpected earnings predict drift in the [d+2, d+60] post-announcement window. Sloan (1996) accrual quality conditions the magnitude: low-accrual-ratio firms (high earnings quality, cash flow exceeds reported earnings) exhibit MORE drift because the market under-reacts to high-quality positive surprises. Bernard-Thomas (1989) day-1 reaction sign confirms the surprise direction is intact (forward guidance has not destroyed the historical surprise).

Long top-quintile Foster-SUE firms whose accruals are below-median (highest earnings quality) and whose day-1 reaction sign matches the SUE sign. Hold equal-weight for [d+2, d+60]. NO regime conditioning. Universe ex Financials/Utilities (Sloan accruals destabilise on banks/REITs). Single-active-window invariant per ticker.

## Why this class is fresh (not a re-test)

Foster SUE was used as a Lasso input feature in the burnt `alt_data` class (4-feature stack with rank_SI, filing_density, etc.). The burnt class evaluated cross-sectional rank (strided 5d Lasso top-N). v3 is structurally orthogonal:

- **Selection rule**: event-driven calendar window vs strided cross-sectional Lasso
- **Feature stack**: SUE is the SOLE ranking signal (no Lasso composition); accruals & Day-1 are HARD gates not features
- **Hypothesis**: PEAD with quality conditioning (Engelberg 2021 mechanism) vs "rank-blind cross-sectional fit" of alt-data class
- **Holding period**: 60 calendar trading days vs 5d strided rebalance
- **Universe**: ex Fin/Util vs the alt_data full Russell-3000-PIT
- **NEW features**: Sloan accruals (fundamental quality), Day-1 market reaction sign

Zen confirmed (gemini-3-pro-preview, high thinking, 2026-05-03): "The correlation of returns generated by an event-driven calendar rebalance vs. a cross-sectional Lasso fit will be near zero."

## Threshold rationale

Program n=19 completed before this test = naive Bonferroni |t| >= 3.30. Zen meta-overfitting argument escalates to **|t| >= 3.50** because:

- 20-th reuse of identical 2024-04-30 → 2026-04-30 holdout
- Implicit cross-class learning from prior 19 FAILs (selection bias on which class to test next)
- Capital-deployment honest accounting requires margin above naive Bonferroni

Phase-dispersion gate alpha_t_range across 5 phases <= 0.5 (per amended v9 gate, more informative than 50pp excess_net_ann legacy gate).

## Known risks (pre-reg auto-pivot triggers)

1. **PEAD compressed by 2024-26 mega-cap regime** (MED-HIGH prob). Cannot mitigate ex-ante. Phase-3 GO/NO-GO at single-phase alpha_t < 1.5.
2. **EDGAR 8-K Item 2.02 timestamp coverage gap** (MED). v3 conservatively defaults to after-hours when hour unknown — eliminates lookahead at cost of one-day delay on every trade. Future enhancement: parse acceptedDateTime from cached SEC submissions JSON.
3. **Sector filter coverage** (MED). v3 ships with empty sic_map (unknown_policy="include") — full Fin/Util exclusion requires SIC ingestion via SEC submissions API or Polygon ticker-details. Until populated, breadth-audit numbers are upper bounds (Fin/Util pollutes cohort).
4. **Burnt-holdout meta-multiplicity** (MED). Threshold escalation 3.50 absorbs. Out-of-sample 2026-Q3 confirmation required if PASS.
5. **Day-1 filter over-restricts breadth** (MED). Phase 2 audit catches; revise pre-reg if mean < 10.

## Anti-patterns explicitly avoided

- No optimizer (no Lasso) — model-free, deterministic
- No hindsight regime conditioning (Mag-7/VIX gates dropped post v1 review)
- No specification search post-holdout-glance (params locked in pre-reg before code)
- No backward-compat shims (event_drift = clean break, no aliases)
- No mock for SUE/accruals in integration tests (real companyfacts JSON fixtures)
